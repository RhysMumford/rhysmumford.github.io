CSS - what's your position?

A little guide to positioning elements in CSS

The five positions of CSS

The position property in CSS is a very important one: it determines where elements are shown on the page and how they interact with the browser window and other elements.

Position What it does Notes
static Elements are positioned according to the normal flow of the page. This is the default position
relative The element has a fixed position on the page, just like static but it can be moved out of position using top, right, bottom and left properties. Other elements respond to the original position of the element, which is kept there like a ghost. -
fixed The element is fixed in place relative to the viewport. It will always stay in place as the page is scrolled and it doesn't leave an impression from its original position. -
absolute The element is fixed in place relative to its nearest element. If there is no element for it to latch onto, it will latch to the page and move along with the page as you scroll. The element it is attached to needs to have a position that is NOT static .
sticky Sticky toggles between a relative and a fixed position. It's positioned in a relative way until a certain scroll position is reached. Then it acts like a fixed element. You have to specify a distance for this to work - either left, right, top or bottom.

How to write the CSS

The syntax for the position of an element is as follows:

{position: static;}
